## Driverpass Project Reflection

The client for this project was DriverPass, they wanted a web based system to help students get ready for their driving exams. The system needed to be able to handle accounts, lessons, scheduling, and practice tests.

What I did well in this project was keeping the general design clear and easy to read. Translating the client's words into a language the devs are able to actually use and then turning the devs words into something the client can understand on the other side.

If I had to redo any part, I'd redo some of the UML from early on. I've learned a lot throughout the class so there are improvements to be made there.

I focused on the user's needs by pulling all of the basic core functions they wanted like the sign up method, scheduling, and testing, then made sure the design showed them clearly. This was important because even if I delivered a beautiful product that didn't match the needs of the customer's request it would be rejected, rightfully.

My approach is to start with teh requirements, build a sort of scaffolding sketch of how it should be, and add detail as I go. I will be using diagrams and constant feedback to improve my systems henceforth on my projects.
