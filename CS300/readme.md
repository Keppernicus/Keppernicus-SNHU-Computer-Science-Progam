### __What was the problem you were solving in the projects for this course?__  
In Project One I needed to analyze how different data structures perform relative to runtime and memory. For Project Two my goal was to build a working program that loads course data and prints the course list in sorted order with their prerequisites.

### __How did you approach the problem?__  
I broke the work into smaller steps. I started with getting file input to work, then moved on to how to store and organize the data. Throughout the course I tested different data structures like vectors, hash tables, and binary search trees to figure out which one made sense for this task. I paid attention to how each one handled insertions, lookups, and memory use.

### __How did you overcome any roadblocks you encountered while going through the activities or project?__  
Most of the roadblocks came from parsing and managing pointers. I ran into bugs with poorly named input and edge cases. To fix them, I slowed down and used simple test files to isolate what was going wrong. I also used debugging tools and printed out values as I went. That helped me catch logic errors as I worked.

### __How has your work on this project expanded your approach to designing software and developing programs?__  
I’ve gotten better at thinking through how data will be used before I decide how to store it. I used to jump straight into using vectors, but now I take the time to ask what kind of operations are going to happen most often. That helps me design code that runs better and is easier to maintain.

### __How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?__  
I started writing smaller functions, naming things more clearly, and adding comments when something wasn't obvious. I also try to keep each piece of code focused on one job. That’s made it easier to go back and fix things or make changes without breaking other parts of the program.
