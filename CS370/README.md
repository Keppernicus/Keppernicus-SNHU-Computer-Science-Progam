
### 1. Briefly explain the work that you did on this project: What code were you given? What code did you create yourself?
- I was given the environment and the scaffolding: TreasureMaze.py, which defines the 8x8 maze, the reward structure, and the game state logic, GameExperience.py, which stores episodes for replay, and notebook helpers for building the neural network, running a test game, and checking whether the agent can win from every free cell.
- What I wrote was the deep Q-learning training algorithm itself. That meant the full training loop from the epsilon-greedy action selection with a decay schedule (start fully random, decay toward a 5% floor as the win rate climbs), Bellman target construction using a periodically synced target network, and based on milestone feedback prioritized experience replay, where transitions are sampled in proportion to their temporal difference error instead of uniformly. In a maze where most moves earn an uninformative -0.04, that focuses training on the rare experiences that actually teach the network something. The agent hit a 100% win rate and passed the full board completion check at epoch 376.

### 2. Connect your learning from throughout this course to the larger field of computer science:
- What do computer scientists do and why does it matter? Computer scientists turn problems into representations a machine can act on. The pirate agent is a small version of that. I didn't tell it the path, I just defined states, actions, and rewards, and the path fell out of the math. That same method is behind everything from speech recognition to fraud detection. This matters because it solves problems at scales no one person (or small group) can eyeball.
- How do I approach a problem as a computer scientist? Decompose, then iterate against evidence. I start with the provided pieces, living examples, or in the worst case the end goal. I then identify exactly what is missing or needed and build the simplest version that could work, and let the training metrics tell me what to change. For this project, when feedback point at the replay sampling, I isolated that mechanism, changed it, and measured the difference. Most of the work is knowing which piece to touch.
- What are my ethical responsibilities to the end user and the organization? I work in privacy and security, so this one isn't abstract for me. To the user I  collect only what the task needs, be honest about what the system does, and remember that a model can learn things you never explicitly gave it. I'll require output on learned information about the customer and let them know, with an option to delete the model. To the organization: surface risks like bias and inexplicable decisions before they ship. A reinforcement learning agent optimizes exactly the reward you define and the responsibility for what that reward ignores belongs to the person who wrote it.
